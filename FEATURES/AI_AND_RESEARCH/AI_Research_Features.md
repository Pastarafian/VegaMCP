# 🤖 AI Swarm & Research Features

VegaMCP is not just an interface for a single LLM; it's a centralized hub for a 10-agent multi-model network that coordinates AI logic natively.

## 1. LLM Output Evaluation (`llm_eval`)
New in v7.2, the `llm_eval` tool provides an automated way to mathematically score, validate, and compare the outputs generated by language models.

### Metrics Computed
- **Relevance:** How related is the answer to the prompt?
- **Coherence:** Is the text logical and natural?
- **Completeness:** Does it cover all aspects of the query?
- **Faithfulness:** Does the claim align with the provided context?
- **Format:** Does it strictly adhere to requested schemas (e.g., JSON)?
- **Toxicity Check:** Is there harmful, biased, or inappropriate content?
- **Hallucination Check:** Does it inject fabricated facts not found in the context?
- **Rubric:** Custom boolean scoring against user-defined criteria.

### Tutorial: Evaluating Model Responses
1. When generating critical system text or code, pass the LLM's raw output to `llm_eval` using the `evaluate` action.
2. Provide the original prompt and any grounding context.
3. The engine will return a JSON object with scores out of 1.0. If `hallucination` scores poorly, the supervising agent can decide to re-prompt the sub-agent or switch models.

## 2. Agent Swarm Orchestration (`swarm`)
VegaMCP runs an internal task queue designed to divide complex work across specialized agents.

### The 10 Specialized Agents
- **`planner`**: Architecture & system design.
- **`coder`**: Writes and patches code.
- **`researcher`**: Scrapes the web and generates knowledge items.
- **`qa`**: Tests functionality and writes Playwright E2E tests.
- **`security`**: Audits code for vulnerabilities.
- **`devops`**: Writes GitHub Actions, Dockerfiles, and manages VPS.
- **`data_analyst`**: Cleans data and executes Python pandas scripts.
- **`designer`**: Converts UI/UX wireframes into React/Tailwind code.
- **`reviewer`**: Final pass audit of code patches (linting/PR reviews).
- **`coordinator`**: Orchestrates pipelines between multiple agents.

### Tutorial: Task Handoff
1. Submit a high-level task to the `swarm` tool via `create_task`.
2. The `coordinator` will decompose it into sub-tasks.
3. It will assign `researcher` to find dependencies, `coder` to implement it, and `qa` to verify it. You can track this async process via the `get_task_status` action.

## 3. Intelligent Model Routing (`route_to_reasoning_model`)
Never rely on a single model. This tool acts as an API gateway that dynamically sends prompts to the best specialized cloud model (DeepSeek R1, GPT-4o, Claude 3.5 Sonnet) or local Ollama model based on the complexity and token budget.
